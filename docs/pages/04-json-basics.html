4 JSON Basics

The root object — everything starts here

{
  "title": "Simple Query Flow",
  "notes": "Project notes here.",
  "nodes": [ ... ],
  "connections": [ ... ],
  "sequence": [ ... ],
  // Optional sections:
  "zones": [ ... ],
  "phases": [ ... ],
  "flows": [ ... ],
  "story": { ... }
}
FieldTypeDescription
titleStringDisplayed at top-left of the canvas
notesStringFreeform project notes (use \n for breaks)
nodesArrayComponents on the canvas
connectionsArrayStatic lines between nodes
sequenceArrayDefault animation steps
zonesArrayInfrastructure boundary boxes
phasesArrayProgressive reveal layers
flowsArrayNamed animation sequences
storyObjectNarrative layer for presentations

Minimal Working Example

{
  "title": "Simple Query Flow",
  "nodes": [
    { "id": "usr", "type": "user", "tag": "external",
      "label": "User", "x": 50, "y": 100, "w": 100, "h": 70 },
    { "id": "db", "type": "database", "tag": "core",
      "label": "SQL DB", "x": 300, "y": 100, "w": 100, "h": 70 }
  ],
  "connections": [ { "from": "usr", "to": "db" } ],
  "sequence": [
    { "from": "usr", "to": "db", "text": "Select * from Users" },
    { "from": "db", "to": "usr", "text": "Return Data" }
  ]
}
Only title, nodes, connections, and sequence are required. Everything else is optional and degrades gracefully.
