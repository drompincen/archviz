7 Sequence & Flows

Animation steps, popup notifications, and named flow sequences

Sequence Step Format

{
  "from": "gw",
  "to": "order-svc",
  "text": "Forward authorized request",
  "status": "ready",
  "phase": "processing",
  "popUp": { "type": "amplify", "msg": "Order confirmed" }
}

TypeStyleUse for
amplifyGreen checkmarkSuccess confirmations

issueRed hexagonErrors or failures

Named Flows

Flows define multiple animation paths through the same architecture. A dropdown appears when flows are present.

"flows": [
  {
    "id": "happy-path",
    "name": "Happy Path — Place Order",
    "phases": ["processing"],
    "sequence": [ ... ]
  }
]

Phase Sync Behavior

When you select a flow whose phases don't include the current phase, the phase slider automatically jumps to the first matching phase. Each flow's sequence is independently filtered by the current phase.
Backwards compatible: The root sequence array is used as "Default Sequence". Flows without a phases property use legacy behavior — shown whenever any of their steps are visible.
